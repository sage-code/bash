Bash Syntax

Bash is a curly bracket language. Unlike all other curly bracket languages are used only for functions. The curly brackets have a secondary meaning. Other blocks have an original design, similar to Ada and Pascal, except blocks are starting and ending with different keywords.

Hello World

This tutorial is based on code snippets we analyze. Next script is just a simple app that print on the console the message "Hello World". This is not much, but we can see several syntax choices that you must notice to start learning. Read this example and notes:

#!/bin/bash
echo "Hello World"

Notes:

  • First line is called "chee bang" and is a special comment;
  • A string is enclosed in double quotes;
  • Second line "echo" is a function call with a parameter;
  • In Bash, we do not enclose arguments in paranthesis;
  • In Bash, end of line optional end with semicolon;

A REPL application is a classic. You display a menu and then you read the input. Depending on the option, you enter a number. Then you create a jump table or a selection statement to execute one or other feature of the APP.

REPL Application

Example

Next app is a demo I have initially created but later replace with a better version. Read the comments to understand better how the application work. You do not have to understand this script, just browse the syntax. Read the notes afterwards.

#bash-repl app
# demo for Bash
echo Bash Demo
echo "-------------------"
echo "0 - exit"
echo "1 - hello"
echo "2 - selection"
echo "3 - for loop"
echo "-------------------"

option=9

function hello()
{
  echo Hello World
}

# submenu demo
function selection()
{
    echo "select:"
    local op="one two three break"
    # demonstrate selection
    select x in $op;
    do
      # demonstrate conditional;
      case $x in
        (break) echo "done"; break;;
        (one)   echo "one";;
        (two|three)  echo "other";;
      esac
    done #select
}

# selection loop
while (($option))
do
  read -p ":>" option
  case $option in
    (0) echo "goodby";;
    (1) hello;;
    (2) selection;;
  esac
done

Notes:

  1. In Bash we can have global scope;
  2. In global scope we can have statements;
  3. The statements can be intrerupted by functions;
  4. After one function is declared, it can be called;
  5. Functions can be called from inside other statements;
  6. Bash do not have switch but select instead;
